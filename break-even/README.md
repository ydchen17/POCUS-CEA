# POCUS Break-Even Analysis

R code accompanying the health economic analysis of point-of-care ultrasound (POCUS) programs in low- and middle-income countries (LMICs).

## Overview

`break_even_analysis.R` computes the **break-even disease burden reduction** required for a POCUS program to be cost-effective, given country-specific cost-effectiveness thresholds and baseline disease burden. Results are stratified by income group and cost scenario (Sub-Saharan Africa vs. global), and a one-way sensitivity analysis is conducted across key cost and structural parameters.

## Directory structure

```
break-even/
└── break_even_analysis.R       # Main analysis script

data/                           # Shared inputs (repo root)
├── thresholds.csv              # Country-level CE thresholds and baseline burden
├── structural_parameters.csv   # Program structural parameters
└── cost_inputs.csv             # Equipment, training, and per-unit costs by scenario

outputs/                        # Generated by the script (not tracked)
```

## Input data

### `data/thresholds.csv`

| Column | Description |
|---|---|
| `country` | Country name |
| `income_label` | World Bank income classification |
| `model_cost_scenario` | Cost scenario applied to this country (`ssa` or `global`) |
| `threshold_label` | CE threshold label (e.g., `1x GDPpc PPP`, `Ochalek low`) |
| `ce_threshold` | CE threshold value (USD/DALY) |
| `baseline_burden` | Baseline disease burden (DALYs per person per year) |

### `data/structural_parameters.csv`

| Column | Description |
|---|---|
| `name` | Parameter name (`throughput`, `useful_life`, `uptake`, `units_per_person`) |
| `base` | Base-case value |
| `low` | Low bound for sensitivity analysis |
| `high` | High bound for sensitivity analysis |

### `data/cost_inputs.csv`

| Column | Description |
|---|---|
| `name` | Cost component (`equipment_cost_package`, `training_cost_package`, `per_unit_cost`) |
| `scenario` | Cost scenario (`ssa`, `global`, or `all`) |
| `base` | Base-case cost (USD) |
| `low` | Low bound for sensitivity analysis |
| `high` | High bound for sensitivity analysis |

## How to run

Requires R with the `dplyr` package installed.

```r
install.packages("dplyr")   # if not already installed
source("break-even/break_even_analysis.R")
```

Run from the **repository root** so the script can locate `data/` and write to `outputs/`. Output CSVs are written to `outputs/`.

## Outputs

| File | Contents |
|---|---|
| `break_even_country_results.csv` | Break-even burden reduction for every country × threshold combination |
| `break_even_summary_by_income_and_threshold.csv` | Median (IQR) break-even relative reduction by income group and threshold |
| `break_even_country_examples.csv` | Detailed results for seven illustrative countries |
| `break_even_sensitivity_summary.csv` | Change in median break-even reduction under low/high parameter bounds |

## Methods summary

The incremental program cost per person screened is:

```
incremental_cost = fixed_cost_per_person + variable_cost_per_person

fixed_cost_per_person  = (equipment_cost + training_cost) / (useful_life × throughput)
variable_cost_per_person = uptake × units_per_person × per_unit_cost
```

The break-even absolute burden reduction is:

```
break_even_burden_reduction = incremental_cost / ce_threshold
```

The break-even **relative** burden reduction (%) is:

```
break_even_relative_reduction (%) = 100 × break_even_burden_reduction / baseline_burden
```
